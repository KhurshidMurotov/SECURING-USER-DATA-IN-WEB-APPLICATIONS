Password Hashing (Django)

Passwords are never stored in plaintext. Django stores PBKDF2 hashes only.

Dummy hash format example: pbkdf2_sha256$600000$demoSalt$demoHashValue

Encryption at Rest (Fernet)

Sensitive profile fields (address, notes) are encrypted before writing to the DB.

Verification steps (local):

  1. Save profile data via UI.
  2. Inspect profiles_userprofile in SQLite.
  3. Stored value differs from plaintext and resembles a Fernet token (gAAAAA...).

CSRF Protection

POST forms include a CSRF token (hidden input csrfmiddlewaretoken).

Missing token leads to HTTP 403 Forbidden.

Brute-Force Protection + CAPTCHA (django-axes)

Login protection flow:

  1. After 3 failed logins, CAPTCHA appears (example: What is 7 + 3?).
  2. If CAPTCHA is solved, 2 more attempts are allowed.
  3. Further failures lock the account for 30 minutes.

Enforcement is handled by django-axes. Lockout UI uses the existing Account Temporarily Locked page.
